# תיקון Rate Limit של Resend

## הבעיה

Resend מאפשר רק **2 בקשות לשנייה**. כששולחים כמה מיילים ברצף, מקבלים שגיאה:
```
Too many requests. You can only make 2 requests per second.
```

## מה תוקן

1. **הוספתי delay של 600ms** בין שליחות מייל
2. **טיפול בשגיאות rate limit** - עכשיו המערכת מטפלת בשגיאות rate limit בצורה יפה
3. **שיפור הלוגים** - עכשיו תראו הודעות ברורות יותר

## איך זה עובד עכשיו

כששולחים מייל למנהל ואז מייל אישור למשתמש:
1. שולח מייל למנהל
2. מחכה 600ms (למניעת rate limit)
3. שולח מייל אישור למשתמש

אם יש rate limit, המערכת:
- לא תקרוס
- תדפיס אזהרה ברורה
- תמשיך לעבוד (המייל למנהל כבר נשלח)

## אם עדיין יש rate limit

אם אתם שולחים הרבה מיילים ברצף, תצטרכו:
- להגדיל את ה-delay (לשנות את `600` ל-`1000` או יותר)
- או לשדרג את התוכנית של Resend

## שינוי ה-delay

אם תרצו לשנות את ה-delay, חפשו ב-`backend/src/services/email.js`:
```javascript
await delay(600); // שנה את 600 ל-1000 או יותר
```
