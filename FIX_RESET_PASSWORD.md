# תיקון כפתור איפוס סיסמה

## מה תוקן

1. **Validation של הקוד** - עכשיו הקוד יכול להיות כל אורך (לא רק 6 ספרות)
2. **ניקוי רווחים** - הקוד מנוקה מרווחים לפני השוואה
3. **הודעות שגיאה בעברית** - כל ההודעות עכשיו בעברית
4. **בדיקת pendingEmail** - המערכת בודקת שיש אימייל לפני שליחה

## איך זה עובד עכשיו

1. משתמש מבקש איפוס סיסמה
2. המערכת שולחת קוד (או מציגה אותו בקונסול אם אין מייל)
3. משתמש מזין את הקוד + סיסמה חדשה
4. המערכת:
   - מנקה את הקוד מרווחים
   - משווה אותו לקוד במסד הנתונים
   - בודקת שהקוד לא פג תוקף
   - מאפסת את הסיסמה

## אם עדיין לא עובד

### בדקו את הלוגים בשרת

כשמבקשים איפוס סיסמה, אמורים לראות:
```
📧 Sending password reset email to ... via Resend...
✅ Password reset email sent to account email: ...
```

### בדקו את הקוד שנשלח

אם הקוד מופיע בקונסול (במקום במייל), העתיקו אותו בדיוק כמו שהוא.

### נסו לבקש קוד חדש

אם הקוד פג תוקף (יותר מ-10 דקות), בקשו קוד חדש.

## אם יש שגיאה

אם אתם מקבלים שגיאה, בדקו:
1. שהאימייל נכון
2. שהקוד נכון (ללא רווחים)
3. שהסיסמה החדשה לפחות 8 תווים
4. שהסיסמאות תואמות
